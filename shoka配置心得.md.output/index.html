# 常见 bug 解决方案

# shoka 配置后显示不全的原因

最主要是 /themes/shoka/_config.yml 里的 menu 那一系列没有取消注释，可以对照着好好操作。

/root/_config.yml 与 /root/themes/shoka/example/_config.yml 对应

/root/themes/shoka/_config.yml 与 /root/themes/shoka/example/_config.shoka.yml 对应

# local 显示正常，域名访问图片无法显示

这个 bug 比较奇怪，我不清楚我这个解决方案是否正确。我是除了 hello world.md 文件以外，又上传了一篇博客，然后全部图片就能正常显示了，仅供参考。

# 每次 hexo d 之后 github page 都要重新设置一下域名

解决方案：在 /root/source 新建 CNAME，里面仅输入域名，如 jy-geeker.top (不要加 http 或者 www)

# 配置解析

# 自定义图库

# 方法 1 借用 GimHoy 图床

Gimhoy 图床 | 无限制永久免费图床 - 微博图床 - 全网 CDN - 永久外链图床 - tuchuang - 图片外链 - 外链相册 - 国内图床 - 论坛图片 - 淘宝图片 | pic.Gimhoy.com

上传图片后记录每个图片链接的最后一串信息（参考 themes/shoka 目录下的_images.yml)，图片要至少 10 张左右。如果在 yml 文件中最后一行多打了 - 则会报错。

用这一图床图片右下角会有水印。不推荐

# 方法 2 借用 PicGo

PicGo 直接用 github 可不可行我不清楚，下面介绍用 PicGo 插件 github-plus 。

  1. 下载方式为 PicGo 的 plugins settings 搜索并下载

  2. githubPlus 的 repo 填写用户名 / 仓库名

  3. branch 注意是 main 还是 master，github 自动生成为 main

  4. token 在 github 的右上角头像点击 settings->Developer settings (最左下角)->Personal access tokens->token (classic)->generate new token->classic->note 自己填 -> 注意勾选 repo 然后滚到最下面 Generate token (发现忘记勾选 repo 时重新回到这界面一定要记得勾选后滚到最下面保存！！！)

  5. 设置代理（不清楚要不要这一步，可能会快一点），以 clash 为例，为 127.0.0.1：7890

# md 文件中的图片无法正常显示

可以参考 github 上的这个脚本，这个脚本的前提条件是你用的图床工具 PicGo 已经配置好了，具体配置说明请看 自定义图库 的方法 2。此外，在运行程序前，请确保 PicGo 已经启动。

JHex-git/MdPicBatchUploader: 基于 PicGo 将 md 文件里所有的本地图像一键批量上传并更新图像链接的 python 脚本 (github.com)

执行程序后，把各路径输入后，它会在你输入的文件路径下生成两个 output 文件，把这两个文件剪贴到 /root/source/_posts/ 下后重新生成博客即可正常显示图片。
